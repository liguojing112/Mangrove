## 内容审核系统重设计

### 问题根源
1. 审核页缩略图逻辑复杂导致渲染失败
2. ResourceManager 所有上传都设 status=0，没区分超级管理员
3. 侧边栏审核菜单只匹配 `ADMIN` 角色，超级管理员看不到入口

### 修复方案（3 处改动）

**1. `AdminLayout.vue` — 审核菜单角色**
- 把 `role: 'ADMIN'` 改成 `role: 'SUPER_ADMIN'`，让超级管理员能在侧边栏看到"内容审核"入口
- 普通管理员看不到审核菜单（他们没权审批）

**2. `ResourceManager.vue` — 上传时根据角色设置状态**
- 导入 `useAuth` 获取当前用户角色
- 超级管理员上传 → `status: 1`（直接已审核）
- 普通管理员上传 → `status: 0`（待审核）
- 资源管理仍然显示全部文件（不加 status 过滤）

**3. `AuditPage.vue` — 简化模板**
- 去掉调试信息
- 缩略图改成简单的文字标签（照片/视频/音乐），不用 `<img>` 标签
- 用 `useAuth` 获取 token

### 流程
- **超级管理员上传** → 直接已审核 → 前台立即可见
- **普通管理员上传** → 待审核 → 超级管理员在审核页通过 → 前台可见
- **资源管理** → 显示全部文件 + 审核状态标签
- **照片/视频页** → `?status=1` 只显示已审核
